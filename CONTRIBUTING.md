# Contribuindo para Between Worlds

Obrigado por contribuir com este projeto! Este documento explica o fluxo de contribuição e as melhores práticas.

## Antes de começar

1. Abra uma issue antes de iniciar mudanças significativas.
2. Verifique as issues existentes para evitar trabalhos duplicados.
3. Use uma branch dedicada para cada melhoria ou correção.

## Fluxo de contribuição

1. Fork do repositório.
2. Crie uma branch local com um nome claro, por exemplo:
   - `feature/nova-funcionalidade`
   - `fix/bug-descricao`
3. Faça commits pequenos e descritivos.
4. Envie um pull request para a branch principal.

## Padrões de commit

- `feat:` para novas funcionalidades
- `fix:` para correções de bugs
- `docs:` para mudanças em documentação
- `chore:` para tarefas de manutenção

- [ ] Código compilou sem erros
